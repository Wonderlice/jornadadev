
# Exercício 1 — Isto é um algoritmo válido?
Para cada situação abaixo, indique se é um algoritmo válido. Se não for, explique por quê
(lembre dos três requisitos: finito, ordenado e sem ambiguidade).

## a. “Some dois números.”
**Resposta:** Não
**Motivo**
É ambíguo, pois não descreve nem a entrada nem a saída. Portanto, não é um algoritmo válido


## b. “Leia um número. Se for positivo, mostre ‘positivo’. Senão, mostre ‘negativo’.”
**Resposta:** Sim
**Motivo**
É um algoritmo válido, pois possui uma sequência ordenada de passos, recebe uma entrada, produz uma saída e termina após exibir o resultado.

## c. “Fique repetindo até acabar.”
**Resposta:** Não
**Motivo**
É um algoritmo inválido, pois não apresenta um fim definido, não atende ao critério de finitude e é ambíguo, já que não especifica o que deve ser repetido nem qual ação deve ser executada em cada repetição.

## d. “Pegue um ingrediente qualquer e cozinhe.”
**Resposta:** Não
**Motivo**
É um algoritmo inválido, pois não apresenta um resultado determinado, não é claro em suas definições e é ambíguo, permitindo diferentes interpretações das instruções.
